# My Side Quests

작고 느슨하게 만든 vibe-coding 프로젝트들을 작업 기록과 회고록처럼 보관하는 저장소입니다.

이곳은 잘 닦인 포트폴리오가 아닙니다. 완성작, 중단작, 실패한 실험, 이상한 밤샘 아이디어가
각각 어떤 작업이었고, 당시 나는 무엇을 생각했고, 지금 돌아보면 무엇이 남는지 적어두는 쪽에
가깝습니다.

`Side Quest`라는 이름은 분위기일 뿐입니다. 기록을 빡빡한 퀘스트 규칙에 맞추기보다, 오래된
작업장을 다시 열어보고 짧게 메모하는 느낌으로 씁니다.

## 작업 목록

아직 보관된 작업이 없습니다.

첫 기록을 추가하면 아래 표를 교체합니다.

| 작업 | 시기 | 한 줄 회고 | 링크 |
|---|---|---|---|
| _empty_ | - | - | - |

## 기록 추가하기

1. [templates/work-log.template.md](templates/work-log.template.md)를 `quests/<year>-<slug>/README.md`로 복사합니다.
2. 기억나는 만큼만 씁니다. 빈칸을 억지로 채우지 않아도 됩니다.
3. 코드 설명보다 당시 작업과 지금의 회고를 우선합니다.
4. 위 `작업 목록` 표에 새 기록을 추가합니다.

## 느슨한 작성 기준

- 하나의 작업은 기본적으로 하나의 `README.md`에 담습니다.
- 스크린샷, 데모, 코드 링크는 있으면 붙이고 없으면 생략합니다.
- 중단된 작업도 괜찮습니다.
- 멋있게 보이게 쓰기보다, 다시 봤을 때 그때의 생각이 떠오르게 씁니다.
- 결론이 꼭 교훈일 필요는 없습니다. "그냥 재밌었다", "이건 다시 안 할 듯"도 기록입니다.
